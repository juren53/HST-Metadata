Unicode normalization is a process that makes sure text is represented in a consistent way in memory, even if there are multiple possible encodings of the same characters.
Why normalization exists

Some Unicode characters can be represented in more than one way:

    Precomposed form: a single code point for a character with an accent.
    Example:

        "é" → U+00E9 (Latin small letter e with acute)

    Decomposed form: a base letter plus a separate "combining" accent mark.
    Example:

        "é" → U+0065 (e) + U+0301 (combining acute accent)

Visually, both look the same → é, but internally, they are different byte sequences.
Unicode Normalization Forms

Unicode defines four standard forms:

    NFC (Normalization Form C – Canonical Composition)

        Prefers the composed form when available.

        "e" + "◌́" → "é" (U+00E9)

    NFD (Normalization Form D – Canonical Decomposition)

        Prefers the decomposed form.

        "é" (U+00E9) → "e" + "◌́"

    NFKC (Compatibility Composition)

        Like NFC, but also converts characters with "compatibility equivalents".

        "①" (circled 1) → "1"

    NFKD (Compatibility Decomposition)

        Like NFD, but also expands compatibility forms.

Example of NFC

import unicodedata

s1 = "e\u0301"        # decomposed form: 'e' + combining acute
s2 = "é"              # composed form: single code point

print(s1 == s2)       # False, different binary representation

s1_nfc = unicodedata.normalize("NFC", s1)
print(s1_nfc == s2)   # True, both now NFC-composed

✅ In short:
NFC normalization converts Unicode text into a consistent, composed form where possible, so that visually identical text has the same binary representation.
